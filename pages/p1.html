Semana 1: Conceptos Clave de QA

¿Qué rol cumple un QA?

Definición

El rol de Quality Assurance (QA) no se limita a encontrar errores, sino que abarca la garantía de calidad a lo largo de todo el ciclo de vida del desarrollo de software. Es un enfoque proactivo que busca prevenir defectos desde la fase de requerimientos.

Responsabilidades Clave

  • - Estrategia de Pruebas: Plan de pruebas y criterios de aceptación.
  • - Revisión de Documentación: Analizar historias de usuario, gestión de riesgos, trazabilidad.
  • - Gestión de Defectos: Documentar, priorizar y dar seguimiento a los bugs encontrados hasta su resolución.
  • - Automatización: Crear y mantener scripts de pruebas automatizadas para optimizar la eficiencia y repetibilidad.

El QA actúa como el responsable de la calidad, trabajando de cerca con desarrolladores y gerentes de producto para asegurar la entrega de un producto robusto y confiable.

¿Qué habilidades debe tener?

Un QA efectivo combina habilidades técnicas sólidas con habilidades interpersonales (soft skills) que le permiten integrarse eficientemente en el equipo de desarrollo.

Habilidades Duras (Hard Skills)
  • - Conocimiento de SQL: Esencial para validar datos directamente en la base de datos.
  • - Manejo de Herramientas de Automatización: Experiencia con frameworks como Selenium, Cypress o herramientas de API Testing (Postman).
  • - Comprensión de Lenguajes de Programación: Lectura básica de código (Java, Python, JS) para entender la lógica del negocio y debugear.
  • - Metodologías de Pruebas: Dominio de diferentes tipos (funcionales, regresión, integración, etc.).
Habilidades Blandas
  • - Pensamiento Analítico: Capacidad para descomponer sistemas y generar escenarios de prueba exhaustivos.
  • - Atención al Detalle: Fundamental para detectar inconsistencias y desviaciones de la especificación.
  • - Comunicación clara: Reportar problemas de manera clara, concisa y objetiva al equipo y a los stakeholders.
  • - Curiosidad y Proactividad: Estar siempre dispuesto a investigar fallas y aprender nuevas tecnologías.

Java: Ventajas y Desventajas

Java es un lenguaje fundamental en el ecosistema de desarrollo empresarial. Entender sus pros y contras es clave para un QA que trabaje en entornos Java/Spring Boot.

Ventajas en Proyectos Empresariales

  • Portabilidad (WORA): "Write Once, Run Anywhere" (Escribe una vez, ejecuta en cualquier lugar) gracias a la JVM.
  • Robustez y Estabilidad: Excelente para sistemas de alto tráfico y aplicaciones empresariales complejas.
  • Seguridad: La JVM proporciona un entorno de ejecución seguro.
  • Ecosistema: Amplia gama de herramientas, librerías y una comunidad madura.

Desventajas en Aplicativos Webs

  • Velocidad de Desarrollo: Puede ser más lento para aplicaciones pequeñas debido a su verbosidad y la necesidad de configuración.
  • Curva de Aprendizaje: Comprender la arquitectura empresarial (e.g., Spring) puede ser un desafío inicial.

Aspectos a validar código

La revisión del código (code review) por parte del QA se centra en la calidad de la implementación y no solo en la funcionalidad. Un buen QA revisa los siguientes puntos:

Calidad del Código (Code Quality)

  • Estándares: ¿El código sigue las convenciones de estilo del equipo (indentación, nombres de variables)?
  • Eficiencia: ¿Hay bucles innecesarios o consultas a la base de datos redundantes?
  • Manejo de Errores: ¿Se manejan las excepciones de forma controlada (try/catch)?
  • Legibilidad: ¿Hay comentarios claros en la lógica compleja? ¿Es fácil para un nuevo desarrollador entender el flujo?
  • Patrones de Diseño:Uso correcto de patrones que aseguren la escalabilidad y modularidad (importante en Java).

Seguridad

  • Validación de Entradas: Verificar que todos los datos provenientes del usuario estén sanitizados para prevenir ataques como XSS o Inyección SQL.
  • Manejo de Secretos: Asegurarse de que las credenciales o tokens sensibles no estén codificados de forma rígida en el código fuente.
  • Manejo de Errores: Evitar que los mensajes de error expongan detalles internos del sistema, bases de datos o rutas de archivo.

Formas de validar código

Existen varios métodos que el equipo de QA o el mismo desarrollador puede utilizar para garantizar la calidad del código fuente antes de su despliegue a producción.

1. Revisión Manual (Peer Review)

  • Intercambio de Conocimiento: Ayuda a que los desarrolladores aprendan del código de sus compañeros.
  • Detectar errores lógicos que las herramientas automáticas podrían pasar por alto.

2. Análisis Estático de Código (SAST)

Herramientas automatizadas (como SonarQube) analizan el código sin ejecutarlo para identificar errores comunes, vulnerabilidades de seguridad (*security bugs*) y problemas de calidad o deuda técnica.

3. Pruebas Automatizadas del Código

  • Pruebas Unitarias:Verifican la funcionalidad de la unidad más pequeña del código (una función o método, como JUnit en Java).
  • Pruebas de Integración: Confirman que diferentes módulos o servicios interactúan correctamente entre sí (bases de datos, APIs).
